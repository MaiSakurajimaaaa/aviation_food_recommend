# 第1章 引言

## 1.1 研究背景及意义

随着全球民用航空业的持续复苏，机上餐食服务已从传统"配餐保障"转变为影响旅客满意度的关键服务触点。2024年我国民航完成旅客运输量7.3亿人次，全国航空配餐企业年配餐量近7亿份。在如此庞大的服务规模下，机上餐食的品质与个性化程度直接影响航空公司的品牌竞争力与运营效率。

然而，传统的机上配餐模式长期沿用"标准化、批量供应"策略，旅客在值机时面对有限的两到三种餐食选项，无法根据个人口味偏好、饮食禁忌或健康需求进行自主选择。这一模式面临三个核心矛盾：其一，旅客饮食需求的多样化与餐食供给的标准化之间的矛盾——不同旅客在口味、餐食类型和健康偏好上存在显著差异，标准化配餐难以覆盖长尾需求；其二，航空公司的成本控制与餐食浪费之间的矛盾——全球航空业每年产生约360万吨客舱和餐食垃圾，其中18%至20%为未食用餐食，过度装载导致的浪费直接转化为燃油消耗与碳排放的增加；其三，旅客期望的实时个性化服务与现有配餐流程信息滞后之间的矛盾——旅客偏好数据在订票、值机、登机各环节碎片化分布，缺乏统一的采集与智能推荐机制。

从理论层面看，个性化推荐系统在电子商务、社交媒体等领域已取得丰硕成果，协同过滤、基于内容的推荐与混合推荐等方法日趋成熟[1][2]。然而，将通用推荐技术迁移到航空餐食这一垂直场景时，面临一系列特有约束：航班餐食供给受航线、舱位与库存的多维约束，推荐结果必须在实际可供应范围内；旅客餐食偏好可能随季节、航线等因素发生短期漂移，静态偏好模型难以捕捉动态变化[3]；新旅客首次乘机缺乏历史行为数据，构成典型的冷启动问题[4]；多餐食航班上需考虑不同餐次的推荐多样性，避免重复推荐[5]。上述约束使得航空餐食推荐成为兼具理论价值与应用价值的研究问题。

从应用层面看，构建一个面向航空旅客的智能美食推荐系统具有多重现实意义：对旅客而言，个性化推荐能提升服务体验，满足多元化饮食需求；对航空公司而言，基于偏好预测的精准配餐能减少餐食浪费、降低运营成本；对行业而言，该系统可为铁路、邮轮等其他交通领域的餐饮服务数字化提供参考范式。

基于此背景，本文设计并实现了一个面向航空旅客的智能美食推荐系统，提出一种融合用户画像匹配、兴趣漂移建模与协同过滤的自适应混合推荐算法，并基于Spring Boot + Vue.js + UniApp技术栈构建了管理端、小程序端与后端服务三端系统，形成"偏好采集—智能推荐—预选确认—航后评分"的完整闭环。

## 1.2 国内外发展现状与发展趋势

### 1.2.1 国内研究概况

国内推荐系统的研究起步虽晚于国外，但近年来在算法改进与行业应用方面均取得显著进展。在协同过滤与混合推荐方面，余江龙等人从多维度综述了个性化推荐系统的研究方法[1]；王素芳等人将协同过滤与内容特征相结合，缓解了数据稀疏场景下的精度下降问题[2]。在兴趣漂移与动态偏好方面，夏翔等人在相似度计算中引入时间权重与兴趣迁移模型[3]；钱聪等人融合时间衰减函数与评论语义相似度，捕捉用户兴趣的渐变性变化[6]；刘超等人通过提取长短期偏好并自适应聚合来处理序列推荐中的偏好漂移[5]。在冷启动问题上，毛骞等人按数据驱动与方法驱动两类进行了系统综述[4]。在协同过滤相似度优化方面，刘昊东等人在余弦相似度中引入Jaccard函数与热门度修正因子[7]，并结合置信度加权策略降低流行偏差对推荐结果的影响。

在餐饮推荐领域，陈南平从用户就餐评论中提取情境信息构建推荐模型[9]；孟佩等人结合协同过滤与多领域数据融合应用于美食推荐[10]；李凯提出了一种融合项目特征与用户偏好的美食推荐方法[11]，为本文提供了直接参考。在航空餐食信息化建设方面，北京航食于2024年上线了旅客餐食预订管理系统，贯通19套系统实现了全链条数字化管理。厦门航空率先推出经济舱免费选餐服务，东航申请了航班配餐量智能预测专利。但将个性化推荐算法深度嵌入航空餐食场景、形成完整闭环系统的工程实践仍较为缺乏。

### 1.2.2 国外研究概况

国外推荐系统研究起步较早，理论体系更为成熟。在基础理论方面，Koren等人提出的矩阵分解技术[16]通过学习用户与物品的隐语义表示，显著提升了推荐精度与可扩展性，为协同过滤奠定了数学基础。在食品推荐领域，Bondevik等人对食品推荐系统进行了系统性综述[17]，指出多数系统依赖显式偏好与物品属性特征进行匹配，而将行为反馈融入推荐闭环的研究仍较为薄弱。

在航空餐食智能化方面，Dwarampudi（2025）提出了基于AI的机上餐食个性化框架[18]，比较了协同过滤、旅客细分、强化学习和神经网络四条技术路径。在产业实践方面，汉莎航空推出"餐盘追踪器"系统，通过AI视觉识别回收餐食的剩余状态以优化配餐量；荷兰皇家航空的TRAYS模型据报可减少63%的餐食浪费；空客公司于2025年推出"Smart Catering"解决方案，通过机载摄像头与云端偏好分析实现智能配餐。总体来看，国外研究更注重端到端的数据闭环，但仍面临真实数据稀缺、个性化与成本平衡等挑战。

### 1.2.3 航空餐食推荐发展历程

航空餐食服务模式大致经历了四个发展阶段。第一阶段（20世纪50年代至90年代）为标准化配餐阶段，机上餐食作为基础飞行保障服务批量供应，不考虑个性化需求。第二阶段（20世纪90年代末至2010年代）为特殊餐食预申请阶段，航空公司开始在购票环节提供清真餐、素食餐等特殊选项，旅客需提前24至48小时申请，但普通旅客仍仅有标准餐可选。第三阶段（2010年代至2020年代初）为数字化预选阶段，航空公司通过App或网站提供餐食预选服务，但推荐逻辑以基于规则的简单排序为主，尚未实现个性化推荐。第四阶段（2020年至今）为智能推荐探索阶段，航空公司开始利用机器学习算法分析旅客偏好与行为信号，AI视觉识别技术被引入餐盘回收环节，形成从"推荐→消费→反馈"的数据闭环，这一阶段的核心特征是推荐系统、需求预测与数据闭环的初步融合，也是本文工作的技术背景。

### 1.2.4 航空餐食推荐发展趋势

综合国内外研究现状与航空餐食服务模式的演进，该领域呈现以下发展趋势。

（1）从简单规则到混合算法。早期主要基于销量排序等简单规则，未来将向多模型融合的混合推荐算法演进，协同过滤、内容匹配与行为反馈的互补融合将显著提升推荐精度[2]。

（2）从静态画像到动态偏好。传统的静态偏好采集正在被时间衰减函数与兴趣漂移检测技术所替代[3][6]，通过对比近期行为与长期历史实现自适应偏好更新。

（3）从单向推荐到闭环反馈。在推荐曝光、点击浏览、预选确认、航后评分等环节埋设行为追踪点，构建从"曝光→交互→选择→评价"的数据闭环，使推荐算法持续迭代优化[11]。

（4）从单次推荐到序列推荐。多餐食航班上需考虑同一旅客前序餐次的选择，避免重复推荐，序列推荐方法将多餐推荐建模为时序决策问题[5]。

（5）从同质化到多层次差异化。引入舱位等级递进的餐食可见规则与差异化推荐策略，实现头等舱、商务舱与经济舱旅客的精准服务。

（6）从数据孤岛到全链路数字化。当前旅客偏好数据在订票、值机、配餐、客舱等系统间分散存储，构建统一的偏好数据平台是全链路数字化的关键方向。

## 1.3 研究内容及结构

本文旨在设计并实现一个面向航空旅客的智能美食推荐系统，提出一种融合用户画像匹配、兴趣漂移建模与协同过滤的自适应混合推荐算法Fused-PMFUP-PRMIDM-AMMBC，将航班供给约束、旅客偏好与行为反馈融入推荐决策。各章节安排如下：第一章为引言，阐述研究背景、国内外研究现状及发展趋势。第二章为相关基础知识，介绍推荐系统基础理论与系统技术选型。第三章为推荐算法原理与实现，阐述三子模型的设计思想、计算公式与代码实现。第四章为系统设计与实现，按照软件工程方法论从需求分析、架构设计到各模块实现进行系统阐述。第五章为系统测试与结果分析，展示推荐算法在不同场景下的推荐效果。第六章为结论与展望，总结研究工作与主要成果，分析不足并展望改进方向。
