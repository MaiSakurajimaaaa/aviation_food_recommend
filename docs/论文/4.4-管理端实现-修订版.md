### 4.4.1 登录与权限管理

管理端登录页采用全屏背景配合居中登录卡片的视觉设计。员工提交账号密码后，后端通过 `POST /admin/employee/login` 验证并签发 JWT 令牌，前端存储于 Pinia 状态及本地缓存，后续请求经由 Axios 拦截器自动携带令牌完成认证。登录后进入主界面，左侧为九大功能模块的垂直导航菜单，右侧为对应的内容展示区。

### 4.4.2 航空总览看板

航空总览页面作为管理端首页，以数据看板形式呈现推荐系统的核心运营指标。页面顶部为六个统计卡片，分别展示推荐总次数、评分均值、热门菜品数和异常用户数。中部为 ECharts 渲染的热门菜品柱状图与菜品分布饼图。底部为推荐异常用户表格，列出偏好未完成或未绑定航班的用户。后端通过 `GET /admin/recommendation/dashboard` 和 `GET /admin/recommendation/top` 接口返回统计数据。

### 4.4.3 航班与旅客管理

航班管理页面采用双模式设计，上半部分为航班搜索与分页列表，下半部分根据是否选中航班自动切换功能模式。

（A）航班旅客模式：选中某个航班后，表格展示该航班关联的旅客列表（含姓名、身份证号、手机号、舱型、偏好完成状态、选餐状态、绑定状态），支持按姓名和身份证号进行本地筛选。管理员可通过"新增用户"对话框选择"新用户录入"或"老用户搜索"两种方式向航班添加旅客，也可对已有旅客信息进行编辑或删除。

（B）全局查询模式：未选中任何航班时，提供跨航班的旅客查询功能——输入航班号、旅客姓名和身份证号即可检索旅客记录，表格额外展示"航班号"和"航线"列。

### 4.4.4 航班餐食绑定

航班餐食中心实现航线-菜品绑定功能。选择航班后，页面展示该航班当前已绑定的菜品列表（含菜品名称、舱位类型、排序）。点击"绑定餐食"弹出对话框，选择菜品并指定舱位类型（头等舱/商务舱/经济舱），通过 `POST /admin/flight/meal` 提交绑定。舱位递进规则在 SQL 查询层生效：查询时以 `cabin_type IN (...)` 条件实现高舱位可见低舱位餐食。航班餐食绑定界面如图 4.3 所示。

### 4.4.5 餐食资源管理

餐食资源中心以表格展示所有菜品，支持按名称和分类筛选。新增菜品时需填写名称、图片、描述、所属分类、餐食类型（`mealType`，四选一）、口味标签（`flavorTags`，六种多选）和库存数量。后端 `POST /admin/dish` 将菜品数据写入 `dish` 表。餐食资源管理界面如图 4.4 所示。

### 4.4.6 评分管理与数据监控

评分管理中心以表格展示所有评分任务，支持按评分状态（PENDING / SUBMITTED / DEFERRED / EXPIRED）和航班号筛选。每条记录显示用户姓名、航班号、评分分数、评分状态、首次可见时间、延期次数与过期时间。管理员可对特定任务执行重开（`POST /admin/recommendation/rating/{id}/reopen`）或手动过期（`POST /admin/recommendation/rating/{id}/expire`）操作。评分管理界面如图 4.5 所示。

用户餐食中心展示旅客的餐食预选记录（含用户姓名、身份证号、航班号、预选状态、菜品名称、口味与数量），支持按航班号统计各菜品的预选需求量分布，辅助配餐决策。后端通过 `GET /admin/user-meal/list` 和 `GET /admin/user-meal/statistics` 提供数据。
